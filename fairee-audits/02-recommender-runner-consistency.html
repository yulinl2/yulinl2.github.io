Audit 2 · Recommender ↔ runner consistency

Systematically enumerated every (notion × miss_pattern × n_cal) combination the recommender can output, then verified that the runner actually dispatches what the recommender promises.

Method: 8-cell audit grid (2 notions × 4 patterns × 2 n_cal regimes) + diff between recommender.recommend_method() output and runner.run_once() diag. Found two real silent-downgrade bugs; both fixed and locked in with regression tests.

Bug 1 (EOO side): fairee_nsc ignored test='bootstrap'

FAIL Silent admission-test downgrade.

Reproduce (pre-fix)

from fairee.algo.method_recommender import recommend_method
r = recommend_method(notion='EOO', n_cal=800, n_test=600, miss_pattern='mild_mnar')
# Pre-fix: r = {method: 'fairee_nsc', test: 'bootstrap', ...}
# Runner dispatch on method='fairee_nsc' ignored cfg['test']
# and silently used Beta admission.

Why it matters

A practitioner reading the recommender's test='bootstrap' would believe they had finite-sample multiplier-bootstrap admission (the W-1 closure for IPW-reweighted cells). They were actually running Beta-quantile admission, which is only asymptotically valid under IPW weights — the W-1 issue the bootstrap was supposed to close.

Fix

Mapped the recommender's output: when n_cal ≤ 1000 and the candidate method is fairee_nsc / fairee_missing, return fairee_bootstrap instead (which the runner DOES dispatch with NSC weights + bootstrap admission via runner.py L198-L200).

Bug 2 (DP side): fairee_dp_nsc ignored test='bootstrap'

FAIL Same bug, mirrored on the DP side.

Reproduce (pre-fix)

from engine.pipelines.runner import run_once
cfg = {"method": "auto", "notion": "DP", "miss_pattern": "mild_mnar",
       "alpha": 0.10, "delta": 0.10, "n": 800, "seed": 0, ...}
r = run_once(cfg)
# Pre-fix: r['diag']['admission_test'] would be 'beta'
# despite the recommender suggesting bootstrap.

Fix

Two changes in runner.py:

  1. The fairee_dp_nsc dispatch now reads cfg['test'] and passes it through to FaiREEDPMissing(test=...) (which already accepted the kwarg).
  2. The 'auto' path now bubbles the recommender's test choice into cfg, alongside the existing buffer bubble.

Why this went undetected

Both code paths were exercised by tests (the dispatch parametrize and the recommender-level tests both passed) — but each test only checked one layer in isolation. Neither verified that the test field output by the recommender survived the runner dispatch. Lesson learned: end-to-end integration tests for the full auto → cfg → run_once → diag pipeline are now in place for both EOO and DP families.
