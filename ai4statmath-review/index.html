Why the reference solution is correct (oracle rationale)
Cover's function-counting theorem gives `C(P,N) = 2 * sum_{k=0}^{N-1} C(P-1,k)` for points in general position with homogeneous (through-origin) separators. For `P=12, N=4` this is `2*(1+11+55+165) = 464`, fraction `464/4096 = 29/256`.
Known edge cases or acceptable variants
`464/4096` accepted only if explicitly reduced to `29/256`. The affine version (sum to `k=N`) is a different, incorrect answer here.
Deterministic verifier / evaluation schema
integer compared to `464`; fraction compared to `Fraction(29,256)`. See `_verify.py` (section P1).
Collision or duplication risk
Cover's theorem is well known; novelty is in the homogeneous (no-bias) framing and the exact `P>2N` regime. Recommend dedup check against any existing "neural coding capacity" items.
Questions for Stage 1 reviewers
Is the "general position + homogeneous separators" specification unambiguous enough to forbid the affine variant?
Suggested next action
collect five real model runs; confirm consistent failure on GPT-5.5 and Claude Opus 4.7.
p2
Observed failure mode
reporting CV instead of CV^2, or assuming Poisson Fano = 1 despite the dead time
Why the reference solution is correct (oracle rationale)
for a renewal process the limiting Fano factor equals CV^2 of the ISI (renewal CLT: Var N(t) ~ sigma^2 t/mu^3, E N(t) ~ t/mu). ISI = Delta + Exp(lambda), so mean = Delta + 1/lambda, var = 1/lambda^2, and F = (1/lambda^2)/(Delta+1/lambda)^2 = 1/(1+lambda*Delta)^2 = 1/1.44 = 25/36.
Known edge cases or acceptable variants
`1/1.44`, `1/(1+lambda*Delta)^2` with numbers substituted.
Deterministic verifier / evaluation schema
compare F to `25/36` (tol 0.001); mean rate `250/3 ~ 83.33 Hz`. Monte-Carlo of long-window counts gives F ~ 0.69 (see `_verify.py`, section P2).
Collision or duplication risk
refractory-Poisson / renewal-Fano is textbook; novelty is the specific dead-time parameterization and the sub-Poisson conclusion. Note conceptual link to Problem 15 (CV^2 also appears) but the tasks differ.
Questions for Stage 1 reviewers
should we also ask for the finite-time Fano curve, or keep it to the asymptotic value?
Suggested next action
collect five real runs; confirm CV-vs-CV^2 confusion is the dominant error.

p15
Observed failure mode
claiming PR <= N instead of PR <= rank, or omitting equality conditions
Why the reference solution is correct (oracle rationale)
with nonzero eigenvalues lambda_1..lambda_r > 0, S1 = sum lambda_i, S2 = sum lambda_i^2. Cauchy-Schwarz on (lambda_i) vs (1,..,1) over the r nonzero eigenvalues gives S1^2 <= r S2 (PR <= rank), equality iff all nonzero eigenvalues equal. Expanding S1^2 = S2 + sum_{i!=j} lambda_i lambda_j >= S2 (PSD => nonnegative cross terms) gives PR >= 1, equality iff exactly one nonzero eigenvalue (rank 1).
Known edge cases or acceptable variants
QM-AM / power-mean / convexity arguments are acceptable substitutes for Cauchy-Schwarz, provided equality conditions are handled.
Deterministic verifier / evaluation schema
this is proof-style; the structural rubric in the LaTeX file is the grader. Numeric sanity sweep over random PSD matrices confirms 1 <= PR <= rank (see `_verify.py`, section P15).
Collision or duplication risk
PR / inverse-participation-ratio bounds are folklore in physics and ML; novelty is the explicit rank-tight statement with both equality conditions as a graded proof. Conceptual link to Problems 02, 07 (PR appears) but this is the only proof-style task.
Questions for Stage 1 reviewers
is the structural rubric specific enough for an agent-as-judge to audit deterministically, or should we add a required lemma (zeros do not affect either trace) as an explicit gate?
Suggested next action
collect real proof attempts; check specifically whether models bound by N vs rank and whether equality conditions are stated.
