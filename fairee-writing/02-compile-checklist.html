2 · Compile checklist

Go from the 27 draft .tex files in _drafts/ to a single submission-ready PDF. Covers required packages, common LaTeX errors, the existing main.tex stub merge, and the short-vs-long flag pattern.

Reading time ≈ 5 min · Reproduce time ≈ 2 min (single PDF build).

One-shot build

cd "FaiREE_Missing_Data/Fairness with missing/Manuscript/"
pdflatex _drafts/outline.tex                  # 1st pass — collect labels
bibtex _drafts/outline                        # resolve \cite
pdflatex _drafts/outline.tex                  # 2nd pass — fill refs
pdflatex _drafts/outline.tex                  # 3rd pass — settle pagination
# Output: _drafts/outline.pdf

The 3-pass cycle is needed because BibTeX runs after the first pdflatex pass; the 2nd pass picks up the .bbl, the 3rd settles pagination cross-references.

Heads up: the _drafts/ path inside outline.tex's \input{} calls assumes cwd = Manuscript/. Running from a different cwd silently breaks. Use the explicit cd above.

Required LaTeX packages

The outline.tex preamble loads:

\usepackage[margin=1in]{geometry}
\usepackage{amsmath, amssymb, amsthm}
\usepackage{bm}
\usepackage{hyperref}
\usepackage{natbib}
\usepackage{algorithm, algpseudocode}
\usepackage{booktabs, multirow}
\usepackage{graphicx}

All of these ship with TeXLive-full / MacTeX-full. If you only have texlive-base, install texlive-latex-extra + texlive-science for algpseudocode + booktabs + natbib.

Theorem environments

outline.tex declares theorem, proposition, lemma, corollary, assumption, remark, definition. The drafts use all of them. If you swap in a venue-specific template (NeurIPS, ICML, JMLR), make sure these are declared in the new preamble — otherwise compile fails with Undefined environment 'theorem'.

Common compile errors and fixes

FAIL Undefined references (?, ??)

Missing label for one of the 5 residual forward-refs (see Guide 1). The PDF will compile but show "?" in place of the section/equation number, and pdflatex will print: 

LaTeX Warning: Reference `prop:pessimistic' on page 4 undefined on input line ...

Resolve by either (a) adding the missing label at the target site, or (b) changing the citing ref to point at an existing label.

FAIL "Missing $ inserted"

Usually an underscore in body text. The drafts use \texttt{} for code identifiers (\texttt{w\_max}) so this should be rare. If you see it, search for the offending line in the file mentioned in the error.

FAIL "Overfull \hbox"

A long URL or unbroken code identifier overflows the column. In body prose, wrap with \sloppy or break the line. In bibliography, biblatex's backend=biber + \hypersetup{breaklinks=true} handles this better than natbib if it becomes a problem.

WARN Headers and footers exceed

Some venues (NeurIPS) require strict page limits including references. Use \bibliographystyle{plainnat} with the venue's .bst file. The findings_summary.tex section can be condensed if needed.

Merging with the existing main.tex stub

The repo has an existing Manuscript/main.tex stub (problem-setup-only) that predates the drafts. Two options:

  1. Replace: cp _drafts/outline.tex main.tex and edit the title/author block. Cleanest if the stub's problem-setup material is already in methodology.tex.
  2. Merge: keep main.tex for §4 (problem setup) and have outline.tex's §4 line read \input{main_problem_setup.tex} (the stub renamed). The outline preamble notes this option in the comment.

Recommended: option 1 (replace). The stub's problem-setup material is shorter and less detailed than what's in methodology.tex + notations.tex.

Short-vs-long version flag pattern

For venues with a short main paper + long appendix (most ML venues), use a conditional flag near the preamble: 

\newif\ifshort
\shorttrue                    % or \shortfalse for long version

% Then in body:
\ifshort
  \input{_drafts/findings_summary.tex}
\else
  \input{_drafts/experiments.tex}
  \input{_drafts/shrinkage_validation.tex}
  \input{_drafts/w_clip_ablation.tex}
  \input{_drafts/bootstrap_admission.tex}
\fi

Use the same pattern to fold detailed assumptions audit into the appendix for tight page limits.

Pre-compile sanity script

Drop this in Manuscript/ as check.sh:

#!/usr/bin/env bash
set -e
cd "$(dirname "$0")"

# 1. Citation consistency
grep -hoP '@\w+\{\K[a-zA-Z0-9_]+' reflist_drafts.bib | sort -u > /tmp/bib.txt
grep -hoP '\\cite[tp]?\{\K[^}]+' _drafts/*.tex | tr ',' '\n' | tr -d ' ' | sort -u > /tmp/cit.txt
broken=$(comm -23 /tmp/cit.txt /tmp/bib.txt)
[ -z "$broken" ] || { echo "BROKEN CITES:"; echo "$broken"; exit 1; }

# 2. Label consistency
grep -hoP '\\label\{\K[^}]+' _drafts/*.tex | sort -u > /tmp/lab.txt
grep -hoP '\\(ref|eqref|cref|autoref)\{\K[^}]+' _drafts/*.tex | sort -u > /tmp/ref.txt
dangling=$(comm -23 /tmp/ref.txt /tmp/lab.txt)
[ -z "$dangling" ] || { echo "DANGLING REFS:"; echo "$dangling"; exit 1; }

# 3. Environment balance
for f in _drafts/*.tex; do
  open=$(grep -oE '\\begin\{[^}]+\}' "$f" | wc -l)
  close=$(grep -oE '\\end\{[^}]+\}' "$f" | wc -l)
  [ "$open" = "$close" ] || { echo "ENV MISMATCH: $f ($open begin / $close end)"; exit 1; }
done

echo "OK — ready to compile."
