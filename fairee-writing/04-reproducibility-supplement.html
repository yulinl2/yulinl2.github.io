4 · Reproducibility supplement

Package the code + result-dir snapshot + headline-figure regeneration scripts as a camera-ready supplement that reviewers can run without contacting the authors.

Reading time ≈ 8 min · Packaging time ≈ 30 min once.

What goes in the supplement

ItemSourceWhy it's in the supplement
Code freeze (tagged release)fairee-experiment recovery branch tagged v0.1.0-paper-submissionReviewer can pip install git+...@tag to get exact code.
Result-dir snapshotexp/results/ (32 dirs, ~5,100 runs)So reviewers can re-derive headline numbers from the CSVs without re-running the benchmarks (which take ~hours).
Locked requirementspip freeze > requirements/locked.txtReproducibility across pip/numpy versions.
Headline-regeneration scriptsSubset of exp/benchmarks/ producing the figures + tables in the paperReviewer runs make reproduce-paper and gets the paper's figures regenerated.
Reviewer READMENew file at supplement rootStep-by-step "to reproduce table N, run script X".

Step 1 — Tag the code freeze

cd fairee-experiment
git checkout claude/recover-dev-intent-eQY0u
git tag -a v0.1.0-paper-submission -m "Paper submission snapshot"
git push origin v0.1.0-paper-submission

Now the code at that tag is immutable. Cite it in the paper: "Code at github.com/yulinl2/fairee-experiment, tag v0.1.0-paper-submission."

Step 2 — Pin requirements

cd fairee-experiment
source .venv/bin/activate
pip freeze > requirements/locked.txt
git add requirements/locked.txt
git commit -m "supplement: pin requirements at paper-submission snapshot"
git push

The locked.txt file is included in the supplement tarball so reviewers can recreate the exact environment.

Step 3 — Result-dir snapshot

cd fairee-experiment
tar -czf supplement-results.tgz \
    exp/results/ \
    FIGURES.md PROOF_MAP.md GLOSSARY.md FAQ.md CHANGELOG.md
# <= 100 MB compressed; fits most supplement-upload limits.

If 100 MB is too large for the venue (NeurIPS caps at 100 MB, ICML at 50 MB), prune to the headline result dirs only: 

tar -czf supplement-results-headline.tgz \

Step 4 — Headline-regeneration scripts

Create a Makefile target that re-derives every figure + table in the paper: 

# In Makefile, add:
.PHONY: reproduce-paper
reproduce-paper:
	# Headline DP-vs-EOO table (Table 1): 30 seeds, ~10 min
	PYTHONPATH=src:exp/methods python exp/benchmarks/dp_vs_eoo_sweep.py \
	    --out exp/results/repro/dp_vs_eoo_tight --reps 30 --n 1500 --alpha 0.05

	# Adult Pareto (Figure 2 / Table 2): 30 runs, ~5 min
	PYTHONPATH=src:exp/methods python exp/benchmarks/adult_benchmark.py \
	    --reps 10 --alpha 0.05 0.10 0.15 --miss-A 0.2 \
	    --methods naive complete_case_eoo fairee_dp fairee_dp_nsc \
	    --out exp/results/repro/adult_dp_full

	# W-2 fix validation (Table 3): 60 head-to-head runs, ~3 min
	PYTHONPATH=src:exp/methods python exp/benchmarks/w2_validation.py \
	    --reps 30 --out exp/results/repro/w2_validation

	# COMPAS DP table (Table 4): 30 runs, ~5 min
	PYTHONPATH=src:exp/methods python exp/benchmarks/compas_benchmark.py \
	    --reps 10 --alpha 0.05 0.10 --miss-A 0.2 \
	    --methods naive fairee_dp_nsc --notion DP \
	    --out exp/results/repro/compas_dp

	# K=3 COMPAS (Table 5): 10 runs, ~2 min
	PYTHONPATH=src:exp/methods python exp/benchmarks/compas_multigroup_dp.py \
	    --reps 10 --out exp/results/repro/compas_multigroup_dp

	# Buffer validation (Figure 3): 20 runs, ~3 min
	PYTHONPATH=src:exp/methods python exp/benchmarks/buffer_validation.py \
	    --reps 20 --n 800 --out exp/results/repro/buffer_validation

	@echo
	@echo 'Re-derived all paper headlines into exp/results/repro/'
	@echo 'Total wall time: ~30 minutes on a single CPU core.'

Step 5 — Reviewer README

Drop this at the supplement root as README_REVIEWER.md:

# FaiREE with Missing Data — Reviewer Reproducibility Supplement

## What's here
- `code/` — fairee-experiment at tag v0.1.0-paper-submission
- `requirements-locked.txt` — exact dependency versions
- `results/` — pre-computed result-dir snapshots from the paper
- `reproduce.sh` — one-shot script to re-derive every paper figure

## Minimum to verify the headline claims
~5 minutes, requires Python 3.10+:

```bash
cd code/
python -m venv .venv && source .venv/bin/activate
pip install -r requirements-locked.txt
PYTHONPATH=src:exp/methods python -m pytest tests/  # should report 120 passing
```

## Full headline reproduction
~30 minutes on a single CPU core:

```bash
make reproduce-paper
# Compare exp/results/repro/ contents to the provided results/ snapshot
```

## Mapping paper claims to result dirs
- **Table 1 (headline DP vs EOO)** ← `results/dp_vs_eoo_tight/summary.csv`
- **Figure 2 (Adult Pareto)** ← `results/adult_dp_full/fig_pareto.png`
- **Table 3 (W-2 fix: 80% → 0% fail rate)** ← `results/w2_validation/`
- **Figure 3 (buffer trade-off)** ← `results/buffer_validation/fig_tradeoff.png`
- **Table 5 (K=3 COMPAS)** ← `results/compas_multigroup_dp/results_agg.csv`

For the complete claim-to-source map, see `FIGURES.md` + `PROOF_MAP.md`
in the code repo.

## How the audit works (in case results disagree)
Every result dict from `run_once(cfg)` carries an `audit_status` field
(ok / warn / fail) and a `audit_warnings` list. See
`src/engine/analysis/audit.py` for the 7 health checks. The
[/fairee-tutorials/03-reading-the-audit.html](https://yulinl2.github.io/fairee-tutorials/03-reading-the-audit.html)
walkthrough explains each.

yulinli@bu.edu
