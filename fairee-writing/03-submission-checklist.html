Venue prep notes

NeurIPS

  • Page limit: 9 pages main + unlimited appendix (8 pages camera-ready). Short-vs-long flag pattern (Guide 2) handles this.
  • Required: paper-checklist appendix (~2 pages). Items relevant to this work: reproducibility (∇), broader impacts, code/data availability (✓ recovery branch is public, MIT-licensed).
  • Anonymous code repo required for submission; anonymous.4open.science handles it.
  • OpenReview format: neurips_2026.sty + author block stripped.

ICML

  • Page limit: 8 pages main + unlimited appendix.
  • Single-blind for the most part (author info visible). Less anonymization needed.
  • Required: \icmltitlerunning{}.

TMLR (Transactions on Machine Learning Research)

  • Open-ended length; rolling submission. Good fit if you want feedback before committing to a venue deadline.
  • Reproducibility criteria are strict; the supplement (Guide 4) is non-negotiable.
  • No anonymization required (it's a journal model).

JMLR

  • Long-form. Up to 50+ pages welcome.
  • Standard jmlr2e.sty.
  • Theorem-heavy presentation works well here — both W-1 and W-2 closures get full proof.

Phase 5 — Final pre-flight

  • CHECK Compile cleanly (no warnings) with the target venue's .sty.
  • CHECK All figure references ~\ref{fig:...} have non-breaking spaces.
  • CHECK Headline number consistency: TLDR / FAQ / abstract / table all say the same numbers (re-run audit 3).
  • CHECK PDF text is selectable / searchable (not all rasterized).
  • CHECK Spell-check pass: aspell -t check main.tex.
  • CHECK Read it cold from page 1 to end — at least once, ideally a day after the last edit.
