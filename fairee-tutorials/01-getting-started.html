1 · Getting started

Install the pipeline, run the 5-method quickstart on synthetic MNAR data, and learn how to read the output.

Reading time ≈ 5 min · Reproduce time ≈ 2 min

Install

git clone https://github.com/yulinl2/fairee-experiment.git
cd fairee-experiment
make setup            # creates a venv + installs base/dev/exp requirements

If you don't have make, the equivalent is:

python -m venv .venv && source .venv/bin/activate
pip install -r requirements/all.txt

Run the quickstart

PYTHONPATH=src:exp/methods python exp/quickstart.py

The script:

  • Generates n = 2000 synthetic rows with mild MNAR on both A and y.
  • Trains a logistic-regression score on a 800-row training split.
  • Runs 5 methods on the 600-row calibration split: naive, complete_case_eoo, fairee_nsc, fairee_dr, fairee_dp.
  • Predicts on the 600-row test split, reports ACC + EOO gap + DP gap + budget status.
  • Shows the audit dict for the fairee_nsc run.
  • Demos the generalization_buffer feature.

Expected output (one seed; numbers will jitter)

Quickstart: n=2000, mild MNAR, alpha=0.1, single seed
Method                        ACC   EOO gap    DP gap      budget?
----------------------------------------------------------------------
naive                       0.722     0.027     0.024  EOO:     in
complete_case_eoo           0.685     0.006     0.008  EOO:     in
fairee_nsc                  0.685     0.006     0.008  EOO:     in
fairee_dr                   0.717     0.127     0.147  EOO:   over
fairee_dp                   0.718     0.021     0.017  DP:     in

How to read this table

  • ACC: test-set accuracy.
  • EOO gap: |TPR_{A=0} − TPR_{A=1}| on the test set.
  • DP gap: |P(ŷ=1|A=0) − P(ŷ=1|A=1)| on the test set.
  • budget?: did the method's targeted gap stay below α = 0.10? "in" = yes, "over" = no.
Why does fairee_dr fail the budget here?

By design. fairee_dr uses doubly-robust cell counts but keeps the Beta-quantile admission test, which silently violates its Bernoulli premise on DR-corrected cells. This is the W-2 issue. The fix is fairee_dr_if_bootstrap (the W-2 closure), which uses an influence-function multiplier bootstrap admission with an explicit finite-sample guarantee.

Health audit for one run

Health audit for the fairee_nsc run:
  status: ok
  - INFO: self-censoring strength 125.36 > 0.30. The data require the
    Li-self-censoring model (not Sadinle NSC); the current NSCDensityRatio
    handles this correctly but the convergence may benefit from more
    BFGS iterations.

INFO messages don't flip the audit status; they're advisories. Only WARN and FAIL do. Full audit catalogue → tutorial 3.

Buffer demo

Buffer demo (test-set guarantee at modest cost):
  recommended_buffer(n_test=600) = 0.0408
  No buffer: acc=0.718, DP=0.017
  Buffer   : acc=0.653, DP=0.002

The buffer subtracts from α at admission time, leaving headroom for the cal-vs-test sampling gap. Default 0; "auto" resolves to 2·√(p(1−p)/n_test). Empirically: 4–5× test-set fail-rate reduction in the typical regime (see buffer_validation).
