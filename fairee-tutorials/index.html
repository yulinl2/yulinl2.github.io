Tutorials

Hands-on walkthroughs for the FaiREE-with-missing-data Python pipeline (code repo, TLDR.md). Each tutorial is reading-time ≈ 5 minutes; reproduce-time ≈ 2–5 minutes.

Audience tier: mid (collaborator picking up the project, practitioner shipping a fair classifier). See GLOSSARY.md for plain-language definitions of any acronym used below.

Tutorial 1 · 5 min
Getting started — install & quickstart
Clone the repo, run exp/quickstart.py, and read its 5-method output. By the end you'll know what the EOO gap, DP gap, and audit status mean for your data.
Tutorial 2 · 5 min
Method selection with method='auto'
Let the recommender pick a FaiREE variant from your data shape + fairness criterion + missingness pattern. Walk through each of the 6 routing branches and what the resulting rationale tells you.
Tutorial 3 · 7 min
Reading the audit
Every FAIL / WARN / INFO the audit can emit, what it means about your run, and what to do about it.

How to use these

Open the tutorial that matches your situation:

  • If you've never run the pipeline before → start with #1.
  • If you have data and want the algorithm to choose for you → #2.
  • If you see audit_status in your results and aren't sure what it means → #3.
