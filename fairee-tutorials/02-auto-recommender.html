2 · Method selection with method='auto'

Let the recommender pick a FaiREE variant + admission test + buffer setting from your data shape, fairness criterion, and best guess at the missingness pattern. Walk through each routing branch and what the resulting rationale tells you.

Reading time ≈ 5 min · Reproduce time ≈ 2 min

One-call usage

from engine.pipelines.runner import run_once

cfg = {
    "method": "auto",
    "notion": "DP",                 # 'DP' | 'EOO' | 'both'
    "miss_pattern": "mild_mnar",    # 'mcar' | 'mar' | 'mild_mnar' | 'heavy_mnar'
    "alpha": 0.10, "delta": 0.10,
    "n": 1500, "seed": 0,
    # ... other generator/data fields
}
r = run_once(cfg)

print(r["method"])                    # e.g. fairee_dp_nsc
for s in cfg["_auto_method_rationale"]:
    print("  -", s)

Sample rationale trail

EOO/heavy_mnar @ n=1500:
  - notion=EOO -> FaiREE-EOO family
  - miss_pattern=heavy_mnar + EOO -> fairee_dr_if_bootstrap
    (W-2 closed; finite-sample DR admission)
  - n_test=450 < 500 -> buffer=0 (would force trivial fallback)

DP/mild_mnar @ n=800:
  - notion=DP -> FaiREE-DP family
  - miss_pattern=mild_mnar -> +NSC reweighting -> fairee_dp_nsc
  - n_cal=240 <= 1000 -> bootstrap admission
  - n_test=320 < 500 -> buffer=0 (would force trivial fallback)

What gets bubbled into cfg

When you pass method='auto', run_once mutates cfg in place to surface the recommender's choices:

  • cfg['_auto_method_rationale']: list of human-readable reasoning steps.
  • cfg['generalization_buffer']: set to the recommended value if not already.
  • cfg['test']: set to 'beta' or 'bootstrap' if not already.

So you can inspect them after the call.

What does NOT get auto-picked

  • Multi-group EOO/DP at K ≥ 3 returns method='FaiREEMultiGroupEOO' or FaiREEMultiGroupDP as a class name with a warning that those aren't in the runner dispatch — call them directly.
  • The AIF360 baselines (Hardt EqOdds, calibrated EqOdds, ROC) are reference points only; the recommender never picks them since they don't carry FaiREE's finite-sample guarantee.

When you should override

Pass an explicit method= in cfg when:

  • You're running a benchmark sweep and want a specific variant for the comparison.
  • You've measured your data's properties (sc_strength, w_max) and want finer control than the recommender's heuristics.
  • You're reproducing a paper's headline number — the paper specifies the method.
